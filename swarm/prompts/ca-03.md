### 2. Content Item Structure

```go
type Meta struct {
    Title       string    `json:"title"`
    Slug        string    `json:"slug"`
    Summary     string    `json:"summary,omitempty"`
    Status      Status    `json:"status"`
    Category    string    `json:"category,omitempty"`
    CreatedAt   time.Time `json:"created_at"`
    UpdatedAt   time.Time `json:"updated_at"`
    PublishedAt *time.Time `json:"published_at,omitempty"`
    PublishURL  string    `json:"publish_url,omitempty"`
}

type Status string
const (
    StatusDraft            Status = "draft"
    StatusQAPassed         Status = "qa_passed"
    StatusPublished        Status = "published"
    StatusSocialGenerated  Status = "social_generated"
    StatusScheduled        Status = "scheduled"
    StatusPosted           Status = "posted"
)

type SocialJSON struct {
    XText        string `json:"xText"`
    LinkedInText string `json:"linkedinText"`
    XPostURL     string `json:"xPostUrl,omitempty"`
    LinkedInURL  string `json:"linkedinPostUrl,omitempty"`
}

type QAJSON struct {
    Checks  []QACheck `json:"checks"`
    Passed  bool      `json:"passed"`
    RunAt   time.Time `json:"run_at"`
}

type QACheck struct {
    Name    string `json:"name"`
    Passed  bool   `json:"passed"`
    Issues  []string `json:"issues,omitempty"`
    Fixes   []QAFix  `json:"fixes,omitempty"`
}

type QAFix struct {
    Original string `json:"original"`
    Fixed    string `json:"fixed"`
    Applied  bool   `json:"applied"`
}
```

### 3. Store Operations

```go
type Store struct {
    ContentDir string // e.g., "content"
}

func NewStore(contentDir string) *Store
```

### 4. State Machine (`state.go`)

Valid transitions:
```
draft → qa_passed          (after QA passes)
qa_passed → draft          (rejected, needs rework)
qa_passed → published      (after publish with approval)
published → social_generated (after social copy generated)
social_generated → scheduled (after scheduling with approval)
scheduled → posted         (after cron fires posts)
```

Invalid transitions must return an error with clear message. When `qa_gate = false`, allow `draft → published` directly.

```go
func ValidTransition(from, to Status, qaGate bool) error
func (s *Store) Transition(slug string, to Status, qaGate bool) error
```

### 5. CLI Commands

**`contentai new <slug> [--title "..."] [--from-idea N]`**
- Creates `content/<slug>/` via Store.Create()
- If `--from-idea N`: reads `content/ideas/` latest batch, seeds article.md with outline N
- Prints confirmation with path

**`contentai list [--status STATUS] [--json]`**
- Lists all content items from Store.List()
- Table format (default): SLUG, STATUS, TITLE, CREATED
- JSON format with `--json`
- Sort by created_at descending

Register these as cobra commands in `cmd/new.go` and `cmd/list.go`.

### 6. Tests

Write tests FIRST:

- `TestCreateAndGet` — create slug → get returns correct meta
- `TestCreateDuplicate` — create same slug twice → error
- `TestList` — create 3 items → list returns all 3
- `TestListFilterByStatus` — create items with different statuses → filter works
- `TestReadWriteArticle` — write article → read returns same content
- `TestReadWriteSocial` — roundtrip social.json
- `TestReadWriteQA` — roundtrip qa.json
- `TestExists` — exists returns true for created, false for missing
- `TestValidTransitionHappyPath` — all valid transitions succeed
- `TestValidTransitionInvalid` — draft→published with qa_gate=true → error
- `TestValidTransitionSkipQA` — draft→published with qa_gate=false → ok
- `TestTransition` — updates meta.json status + updated_at
- `TestSlugValidation` — slugs with spaces/special chars → rejected
- `TestMetaSerialization` — meta.json roundtrip preserves all fields including nullable times

Target: 90% coverage on `internal/content/`.
