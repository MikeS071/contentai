# ca-02: LLM Client — Pluggable Provider with Context Assembly

## Context
You are building the LLM client for **ContentAI** — a Go CLI tool. This package handles all AI interactions: sending prompts with assembled context, streaming responses, and supporting multiple providers.

Read `SPEC.md` in the repo root for full project context.

### 2. Interface

```go
type Message struct {
    Role    string `json:"role"`    // system | user | assistant
    Content string `json:"content"`
}

type Request struct {
    Messages    []Message
    Model       string
    Temperature float64
    MaxTokens   int
    Stream      bool
}

type Response struct {
    Content    string
    TokensIn   int
    TokensOut  int
    FinishReason string
}

type StreamChunk struct {
    Content string
    Done    bool
    Error   error
}

type LLMClient interface {
    Complete(ctx context.Context, req Request) (*Response, error)
    Stream(ctx context.Context, req Request) (<-chan StreamChunk, error)
    Name() string
}

func NewClient(provider, model, apiKey, baseURL string) (LLMClient, error)
```

### 3. OpenAI Provider (`openai.go`)
- Calls `POST /v1/chat/completions`
- Supports `base_url` override for proxies/self-hosted
- Handles streaming via SSE parsing
- Extracts token usage from response
- Error handling: rate limits (429 → retry with backoff), auth errors (401), model errors

### 4. Anthropic Provider (`anthropic.go`)
- Calls `POST /v1/messages`
- Maps to Anthropic's message format (system prompt separate)
- Handles streaming via SSE
- Error handling same pattern as OpenAI

### 5. Context Assembly (`context.go`)

The key differentiator — builds rich context for every LLM call:

```go
type ContentContext struct {
    VoiceProfile string   // content/voice.md
    Blueprint    string   // content/blueprint.md
    Examples     []string // content/examples/*.md
    Source       string   // additional source material
    Conversation string   // OpenClaw conversation history (if enabled)
    CustomRules  string   // QA rules file content
}

// AssembleContext reads files from content dir and builds ContentContext
func AssembleContext(contentDir string, opts ...ContextOption) (*ContentContext, error)

// BuildMessages creates the message array for an LLM call
// System message = voice + blueprint summary
// User message = task prompt + source + examples
func (cc *ContentContext) BuildMessages(taskPrompt string, maxTokens int) []Message
```

Context assembly must:
- Truncate intelligently if context exceeds model limits (drop oldest examples first, then truncate conversation history, never truncate voice profile)
- Include voice.md in EVERY call (non-negotiable)
- Include blueprint.md core ideas section in every content-generation call
- Include example articles for draft/QA calls

### 7. Tests

Write tests FIRST:

- `TestNewClientOpenAI` — creates OpenAI client with valid params
- `TestNewClientAnthropic` — creates Anthropic client
- `TestNewClientInvalid` — unknown provider → error
- `TestOpenAIComplete` — mock HTTP server, verify request format, parse response
- `TestOpenAIStream` — mock SSE server, verify chunks received in order
- `TestOpenAIRateLimitRetry` — mock 429 → 200, verify retry happened
- `TestAnthropicComplete` — mock HTTP, verify Anthropic message format
- `TestAssembleContext` — temp dir with voice.md + blueprint.md + examples → correct ContentContext
- `TestAssembleContextMissingVoice` — no voice.md → error (non-negotiable)
- `TestBuildMessages` — verify system/user message structure
- `TestBuildMessagesTruncation` — oversized context → examples dropped first, voice preserved
- `TestResolveAPIKey` — `echo test-key` → cached result
- `TestResolveAPIKeyCache` — second call doesn't shell out again
