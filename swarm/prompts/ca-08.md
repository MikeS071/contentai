# ca-08: `contentai draft` — Article Drafting

## Context
Phase 1+2 code merged. You have: config, LLM client, templates, content store, init wizard, KB.

Read existing code in `internal/` before writing.

## What to Build

`contentai draft <slug> [--source file.md] [--interactive]`

### Prompt Chain
1. **Creative Thought Partner** — mines source material for insights using 4 drivers: pattern spotting, paradox hunting, naming the unnamed, contrast creation
2. **Blog Writer** — takes insights + voice profile → 500-1000 word article

### Context Assembly (every LLM call includes)
- `content/voice.md` — brand voice
- `content/blueprint.md` — intellectual blueprint (core ideas, mission)
- `content/examples/*.md` — style reference articles
- `content/<slug>/meta.json` — title, topic
- `content/<slug>/article.md` — existing draft (if refining)
- `--source` material
- Conversation history (if OpenClaw enabled)

### Interactive Mode
`--interactive`: CLI engages in back-and-forth conversation to mine insights before writing. User types responses, LLM asks follow-up questions using Creative Thought Partner prompt. When ready, synthesizes into article.

### Files
- `internal/draft/drafter.go` — draft orchestration
- `internal/draft/context.go` — context assembly from all sources
- `internal/draft/drafter_test.go` — tests FIRST
- `cmd/draft.go` — cobra command

### Tests (write FIRST)
- `TestDraftAssemblesFullContext` — voice+blueprint+examples+meta all included
- `TestDraftChainsTwoPrompts` — Creative Thought Partner runs, then Blog Writer
- `TestDraftSavesArticle` — article.md written to content/<slug>/
- `TestDraftRefinesExisting` — existing article.md included in context for refinement
- `TestDraftWithSource` — --source file content included
- `TestDraftInteractiveMode` — mock stdin/stdout conversation flow
- `TestDraftUpdatesMetaStatus` — meta.json status updated to "drafted"

Target: 75% coverage on `internal/draft/`.
