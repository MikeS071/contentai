## What to Build

The `contentai kb` command family for managing influential content:

### Subcommands

```
contentai kb add-feed <url>           # add RSS/Atom feed
contentai kb add-feed --opml file.opml # import OPML
contentai kb list-feeds                # show tracked feeds
contentai kb sync                      # fetch latest from all feeds
contentai kb add-note <path>           # add raw material (transcript, notes)
contentai kb search "topic"            # search across KB content
```

### Storage Layout
```
content/kb/
├── feeds.toml              # feed registry
├── blogs/
│   └── <domain>-<hash>/    # per-feed dir
│       ├── feed.json       # feed metadata (title, url, last_sync)
│       └── <date>-<slug>.md # individual posts as markdown
└── notes/
    └── <filename>.md       # user-added notes/transcripts
```

### RSS/Atom Parser
- Parse RSS 2.0 and Atom 1.0 feeds
- Extract: title, link, published date, content/description
- Convert HTML content to markdown (use `html.UnescapeString` + simple tag stripping, no heavy deps)
- Handle relative URLs in feeds
- Store as clean markdown with YAML frontmatter (title, url, date, feed)

### OPML Import
- Parse OPML XML → extract feed URLs + titles
- Add each feed to registry

### Feed Sync
- For each registered feed: fetch, parse, save new posts since last sync
- Track `last_synced_at` and `last_post_date` per feed
- Skip already-downloaded posts (match by URL)
- Timeout: 30s per feed, continue on individual feed failure

### Search
- **Primary:** keyword search across all KB markdown files (title + content)
- **Optional enhancement:** if LLM client available, use embeddings for semantic search
- Return ranked results with title, snippet, source path
- `--limit N` flag (default 10)

## Constraints
- RSS/Atom parsing: stdlib `encoding/xml` ONLY, no `gofeed` or similar
- HTTP: stdlib `net/http` with configurable timeout

- Files ≤400 lines, no new external deps
