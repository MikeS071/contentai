# ca-09: `contentai qa` — Quality Assurance

## Context
Phase 1 code merged. You have: config, LLM client, content store.

## What to Build

`contentai qa <slug> [--auto-fix] [--approve]`

### 7 Built-in Checks
1. **no_secrets** — scan for API keys, passwords, tokens (regex patterns)
2. **accuracy** — flag unverified claims (LLM-assisted)
3. **voice_consistency** — compare against voice.md (LLM-assisted)
4. **dash_cleanup** — remove unnecessary mid-sentence dashes
5. **dedup** — flag repeated ideas/sentences
6. **reading_level** — flag overly complex passages
7. **length** — warn if outside 500-1000 word target

### Auto-fix Flow
If `--auto-fix`: LLM rewrites flagged issues → presents diff → user approves/rejects each fix.

### Approval Flow
`--approve`: marks content as QA-passed in meta.json. Required before publish.

### Output
- `content/<slug>/qa.json` — results per check (pass/fail/warning, details)
- Console: colored summary
