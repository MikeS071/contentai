# ca-14: OpenClaw Integration

## Context
Final integration phase. All other packages exist.

## What to Build

### Conversation History Reader
- Read OpenClaw workspace memory files (`memory/*.md`)
- Read conversation transcripts if available
- Extract relevant snippets for idea generation context
- Respects `openclaw.enabled` and `openclaw.workspace` config

### Memory Search
- Query memd (localhost:7457) for relevant context
- Fallback: grep workspace memory files

### SKILL.md
Create `skill/SKILL.md` for OpenClaw skill discovery:
```yaml
name: contentai
description: AI-powered content creation, publishing, and social scheduling
commands: [init, kb, ideas, draft, qa, hero, publish, social, schedule, post]
```

### Workspace Snippets
- `skill/AGENTS-SNIPPET.md` — append to AGENTS.md
- `skill/TOOLS-SNIPPET.md` — append to TOOLS.md
- `skill/MEMORY-ENTRY.md` — memd entry

### install.sh
`contentai install --openclaw` — copies SKILL.md, appends snippets to workspace files if not already present.

### Files
- `internal/openclaw/reader.go` — conversation + memory reader
- `internal/openclaw/reader_test.go` — tests FIRST
- `skill/SKILL.md`, `skill/AGENTS-SNIPPET.md`, `skill/TOOLS-SNIPPET.md`, `skill/MEMORY-ENTRY.md`
- `cmd/install.go` — install --openclaw command

### Tests (write FIRST)
- `TestReadConversationHistory` — reads memory/*.md files
- `TestReadMemorySearch` — queries memd endpoint (mock HTTP)
- `TestFallbackGrepSearch` — works without memd
- `TestDisabledOpenClaw` — returns empty when disabled
- `TestInstallAppends` — snippets appended without duplication
