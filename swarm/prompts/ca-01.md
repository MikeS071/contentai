### 2. Config Loader (`internal/config/`)
Parse `contentai.toml` using a TOML library (pelletier/go-toml or BurntSushi/toml).

```go
type Config struct {
    Project      ProjectConfig
    LLM          LLMConfig
    LLMDraft     *LLMOverride   // [llm.draft]
    LLMQA        *LLMOverride   // [llm.qa]
    Images       ImagesConfig
    Publish      PublishConfig
    Social       SocialConfig   // map of platform configs
    Schedule     ScheduleConfig
    OpenClaw     OpenClawConfig
    QA           QAConfig
}

type ProjectConfig struct {
    Name              string `toml:"name"`
    ContentDir        string `toml:"content_dir"`        // default "content"
    DefaultPublishGate bool  `toml:"default_publish_gate"` // default true
    DefaultSocialGate  bool  `toml:"default_social_gate"`  // default true
    QAGate            bool   `toml:"qa_gate"`             // default true
}

type LLMConfig struct {
    Provider   string `toml:"provider"`    // openai | anthropic | custom
    Model      string `toml:"model"`
    APIKeyCmd  string `toml:"api_key_cmd"` // shell command to get API key
    BaseURL    string `toml:"base_url"`    // optional
}

type LLMOverride struct {
    Model       string  `toml:"model"`
    Temperature float64 `toml:"temperature"`
}

type ImagesConfig struct {
    Provider     string `toml:"provider"`      // openai | stability
    Model        string `toml:"model"`
    APIKeyCmd    string `toml:"api_key_cmd"`
    Size         string `toml:"size"`          // default "1792x1024"
    TitleOverlay bool   `toml:"title_overlay"` // default true
}

type PublishConfig struct {
    Type       string            `toml:"type"`        // http | static
    URL        string            `toml:"url"`
    AuthHeader string            `toml:"auth_header"` // default "Authorization"
    AuthCmd    string            `toml:"auth_cmd"`
    AuthPrefix string            `toml:"auth_prefix"` // default "Bearer "
    FieldMap   map[string]string `toml:"field_map"`
    Static     StaticPublishConfig
}

type StaticPublishConfig struct {
    OutputDir string `toml:"output_dir"`
    Format    string `toml:"format"` // markdown | html
}

type ScheduleConfig struct {
    Timezone    string   `toml:"timezone"`     // default "UTC"
    Days        []string `toml:"days"`         // default Mon-Fri
    WindowStart string   `toml:"window_start"` // default "09:00"
    WindowEnd   string   `toml:"window_end"`   // default "09:30"
}

type OpenClawConfig struct {
    Enabled        bool   `toml:"enabled"`
    Workspace      string `toml:"workspace"`
    ChannelHistory bool   `toml:"channel_history"`
    MemorySearch   bool   `toml:"memory_search"`
}

type QAConfig struct {
    RulesFile string `toml:"rules_file"`
    AutoFix   bool   `toml:"auto_fix"` // default true
}
```

Functions:
- `Load(path string) (*Config, error)` — parse TOML file
- `Default() *Config` — sensible defaults
- `(c *Config) Validate() error` — required fields, valid values
- `ExecuteKeyCmd(cmd string) (string, error)` — run shell command, return trimmed stdout (for `api_key_cmd`)

### 4. Tests (`internal/config/config_test.go`)

Must write tests FIRST, then implement:

- `TestLoadValidConfig` — full TOML with all sections → parses correctly
- `TestLoadMinimalConfig` — only `[project]` section → defaults applied
- `TestLoadInvalidTOML` — malformed file → error
- `TestValidateRequiredFields` — missing project.name → validation error
- `TestDefaults` — Default() returns expected values
- `TestExecuteKeyCmd` — `echo test-key` → returns "test-key"
- `TestExecuteKeyCmdFailure` — invalid command → error
- `TestValidateProvider` — invalid LLM provider → error
- `TestValidateScheduleDays` — invalid day name → error
