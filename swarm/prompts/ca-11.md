# ca-11: Publisher Adapters + `contentai publish`

## Context
Phase 1 code merged. You have: config, content store.

## What to Build

`contentai publish <slug> [--approve] [--dry-run]`

### Publisher Interface
```go
type Publisher interface {
    Publish(ctx context.Context, item PublishItem) (PublishResult, error)
}
type PublishItem struct {
    Title, Slug, Content, Summary string
    ImageURL, ImagePath string
    Meta map[string]any
}
type PublishResult struct {
    URL string
    ID  string
}
```

### HTTP Adapter
- POST to configurable endpoint with configurable field mapping
- Auth: Bearer token via `api_key_cmd`, or custom header
- Field map in config: `{"title": "title", "body": "content", "slug": "slug"}`
- Response: extract URL from configurable JSON path

### Static Adapter
- Write to output directory as `<slug>/index.html` (from markdown)
- Copy images alongside
- Generate simple HTML wrapper

### Approval Gate
- `--approve` required to actually publish. Without it: dry-run showing what would be sent
- Meta.json status must be `qa_passed` before publish allowed
- After publish: status → `published`, URL stored in meta.json
