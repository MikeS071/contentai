# ca-13: Social Adapters — X + LinkedIn + Scheduling

## Context
Depends on ca-12 (social copy gen). You have: social.json with generated copy.

## What to Build

### Social Adapter Interface
```go
type SocialPoster interface {
    Post(ctx context.Context, post SocialPost) (PostResult, error)
}
type SocialPost struct {
    Text, ImageURL, ImagePath, ArticleURL string
    Platform string // "x" or "linkedin"
}
```

### X Adapter (OAuth 1.0a)
- Post tweet via X API v2
- Attach image (upload media endpoint first, then tweet)
- Auth via `api_key_cmd` for all 4 OAuth tokens

### LinkedIn Adapter (OAuth 2.0)
- Post via LinkedIn API (ugcPosts or shares)
- Attach image
- Auth via `api_key_cmd` for access token

### Scheduling
`contentai schedule <slug> [--slot next|YYYY-MM-DD]`
- Posting calendar: `content/posting-calendar.json`
- Configurable days + time window from config
- `contentai post --check` — cron calls this, posts any due slots

### HARD RULE: No auto-posting without explicit approval, ever.
`contentai post <slug> --confirm` required. `--check` only fires pre-approved scheduled slots.
